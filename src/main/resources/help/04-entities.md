# Section 4: Entity & Data Model

All entities are JPA-annotated classes in `src/main/java/org/automatize/status/models/`.
No DTOs are used — entities are used directly in service and controller layers.
All primary keys are `UUID` generated by PostgreSQL's `gen_random_uuid()`.

## 37. `Tenant` Entity

**Table:** `tenants`

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Auto-generated UUID |
| `name` | String | NOT NULL, UNIQUE | Tenant display name |
| `createdBy` | String | | Audit: who created |
| `createdDate` | LocalDateTime | | Audit: when created |
| `lastModifiedBy` | String | | Audit: who last modified |
| `lastModifiedDate` | LocalDateTime | | Audit: when last modified |

**Relationships:**
- `organizations` → `List<Organization>` (OneToMany, mapped by `tenant`)

**Business rules:**
- Each tenant is fully isolated — no data crosses tenant boundaries
- The `SUPERADMIN` role operates across all tenants
- Regular users are always bound to exactly one organisation, which belongs to one tenant

## 39. `User` Entity

**Table:** `users`

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Auto-generated UUID |
| `username` | String | NOT NULL, UNIQUE | Login username |
| `email` | String | NOT NULL, UNIQUE | Email address |
| `password` | String | NOT NULL | BCrypt hashed password |
| `role` | String | NOT NULL | `ADMIN`, `MANAGER`, `USER`, or `SUPERADMIN` |
| `enabled` | Boolean | NOT NULL | Soft delete / account disable |
| `organization` | Organization | FK nullable | Null for `SUPERADMIN` |
| `createdBy` | String | | Audit |
| `createdDate` | LocalDateTime | | Audit |
| `lastModifiedBy` | String | | Audit |
| `lastModifiedDate` | LocalDateTime | | Audit |

**Roles:**
- `SUPERADMIN` — cross-tenant access, no organisation binding, must select context before operating
- `ADMIN` — full CRUD within their organisation
- `MANAGER` — manage platforms and apps, cannot manage users
- `USER` — read-only access

**Password storage:** BCrypt with default cost factor (10). Plain-text passwords are never stored or logged.

## 41. `StatusApp` Entity

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | |
| `name` | String | NOT NULL | Application name |

## 45. `StatusIncidentComponent` Entity

**Table:** `status_incident_components`

Junction table linking incidents to affected components.

| Field | Type | Constraints |
|-------|------|-------------|
| `incident` | StatusIncident | FK, part of composite PK |
| `component` | StatusComponent | FK, part of composite PK |

**Why a junction table?** A single incident can affect multiple components simultaneously (e.g., a network outage affecting both the API and the CDN component).

## 48. `StatusUptimeHistory` Entity

**Table:** `status_uptime_histories`

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | PK |
| `app` | StatusApp | FK — the tracked application |
| `date` | LocalDate | The day this record covers |
| `uptimePercentage` | Double | Calculated uptime % for the day |
| `totalChecks` | Integer | Total health checks run that day |
| `successfulChecks` | Integer | Health checks that returned success |
| `createdBy/Date` | | Audit |

**Calculation:** `uptimePercentage = (successfulChecks / totalChecks) * 100`

The `UptimeHistoryService` aggregates raw health check results into daily summaries. The `trigger-daily` endpoint can be called manually to backfill history.

|-------|------|-------------|

## 56. `HealthCheckSettings` Entity

**Table:** `health_check_settings`

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | PK |
| `app` | StatusApp | FK nullable — app-level settings |
| `component` | StatusComponent | FK nullable — component-level settings |
| `enabled` | Boolean | Whether health checks are active for this entity |
| `intervalSeconds` | Integer | Check frequency override |
| `timeoutSeconds` | Integer | HTTP timeout override |
| `expectedStatusCode` | Integer | Expected HTTP response code (default: 200) |
| `organization` | Organization | FK |
| `createdBy/Date` | | Audit |

**Hierarchy:** Component settings override app settings, which override global `application.properties` defaults. The `HealthCheckSettingsService` resolves the effective settings using this hierarchy.
